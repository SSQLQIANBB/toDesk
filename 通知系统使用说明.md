# 通知系统使用说明

## 功能概述

ToDesk 应用现已集成完整的通知系统，包括桌面通知、声音提醒和未读消息徽章功能。

## 主要功能

### 1. 桌面通知

应用支持以下类型的桌面通知：

#### 1.1 私聊消息通知
- 当您收到新的私聊消息时，系统会显示桌面通知
- 通知内容包括发送者姓名和消息内容
- 点击通知可直接跳转到对应的聊天窗口

#### 1.2 群组消息通知
- 群组中有新消息时会收到通知
- 显示群组名称、发送者和消息内容
- 点击通知可直接进入该群组聊天

#### 1.3 来电通知
- 收到视频通话、语音通话或屏幕共享邀请时
- 显示来电者信息和通话类型
- 支持直接从通知中接听

#### 1.4 群组邀请通知
- 收到群组邀请时会显示通知
- 显示邀请者和群组信息
- 可直接从通知中查看详情

#### 1.5 离线消息通知
- 登录后收到离线消息
- 显示离线消息数量
- 点击可查看详细消息列表

### 2. 声音提醒

- 新消息到达时播放提示音
- 可在个人中心独立开关声音功能
- 不同类型的通知可使用不同音量

### 3. 未读消息徽章

- 在用户列表中显示未读消息数量
- 红色徽章标记，支持最多显示99+
- 选择联系人后自动清除未读标记
- 窗口可见时自动清零

## 使用指南

### 首次使用

1. **授权通知权限**
   - 首次访问应用时，系统会请求通知权限
   - 请点击"允许"以启用桌面通知功能
   - 如果拒绝，可在个人中心重新请求

2. **配置通知设置**
   - 进入"个人中心" → "通知设置"
   - 根据个人需求配置各项通知选项

### 通知设置

在个人中心的"通知设置"选项卡中，您可以配置：

#### 基本设置
- **桌面通知开关**：启用/禁用所有桌面通知
- **声音提醒开关**：启用/禁用消息提示音
- **消息预览**：是否在通知中显示消息内容

#### 通知类型设置
可以单独控制每种类型的通知：
- 私聊消息
- 群组消息
- 来电通知
- 群组邀请

#### 测试功能
- 点击"发送测试"按钮可测试通知是否正常工作

### 智能通知

#### 窗口聚焦检测
- 当应用窗口处于前台时，不会显示桌面通知（避免打扰）
- 正在查看的聊天窗口收到消息时不会弹出通知
- 窗口重新聚焦时自动清除未读标记

#### 通知去重
- 同一联系人的连续消息会合并通知
- 使用 `tag` 机制避免重复通知

## 常见问题

### Q1: 为什么没有收到通知？

**可能原因：**
1. 未授予通知权限
2. 通知功能被关闭
3. 浏览器阻止了通知
4. 系统勿扰模式已开启

**解决方法：**
1. 检查个人中心的通知设置
2. 检查浏览器通知权限设置
3. 检查操作系统通知设置
4. 使用"发送测试"功能排查

### Q2: 如何关闭某些类型的通知？

进入"个人中心" → "通知设置" → 关闭对应类型的通知开关

### Q3: 通知声音太大/太小？

目前声音音量默认为 50%，您可以：
1. 调整系统音量
2. 在个人中心完全关闭声音提醒

### Q4: 为什么有时通知会延迟？

可能原因：
1. 网络延迟
2. 浏览器后台节流
3. 系统性能限制

建议保持应用窗口在前台以获得最佳体验。

### Q5: 如何重新请求通知权限？

如果之前拒绝了通知权限：
1. 进入"个人中心" → "通知设置"
2. 点击"请求权限"按钮
3. 或者在浏览器地址栏点击锁图标，重置权限

## 隐私说明

- 所有通知仅在您的设备本地显示
- 通知内容不会被上传到任何服务器
- 您可以随时关闭通知功能
- 通知设置仅保存在本地浏览器中

## 技术实现

### 使用的 API
- Web Notifications API
- Web Audio API (用于提示音)
- localStorage (用于保存设置)

### 通知服务架构
```
NotificationService (单例模式)
├── 权限管理
├── 通知显示
├── 声音播放
├── 设置管理
└── 事件处理
```

- `client-vue/src/views/groupChat.vue` - 群聊集成
- `client-vue/src/views/profile.vue` - 设置界面
