# 群组媒体与移动端改造实施计划

1. 建立 Vitest、Vue Test Utils 和 jsdom 测试环境，只增加本任务所需配置。
2. 先新增失败测试，覆盖 Redis 会话状态、固定频道、迟到成员状态恢复和结束恢复。
3. 先新增失败测试，覆盖远端成员去重、断线重连、退出清理及视频 `srcObject` 绑定。
4. 先新增失败组件测试，覆盖双方本地/远端视频、群聊按钮和屏幕共享按钮状态。
5. 先新增失败布局测试，覆盖手机和桌面视口的主要页面、弹窗及媒体区域。
6. 实现 Redis 群组会话服务和独立媒体房间，保留原事件兼容。
7. 实现前端群组会话状态同步服务，并接入群组列表和群聊页。
8. 实现稳定 `userId` 的远端成员管理和单向 offer 建连流程，修复视频及屏幕流绑定。
9. 完成全页面响应式样式，重点处理固定宽度、侧栏、弹窗、控制区和媒体网格。
10. 逐组运行测试并修复，再执行完整测试、类型检查和生产构建。

实施过程中不创建提交，不删除原有注释；仅在核心状态转换和连接生命周期处增加必要注释。
