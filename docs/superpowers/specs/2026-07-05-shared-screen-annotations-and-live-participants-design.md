# 屏幕共享标注同步与实时参与状态设计

## 背景与根因

当前 `ScreenAnnotation.vue` 只在浏览器本地维护绘图历史，`groupScreen.vue` 中的“开启标注”按钮也只切换 `showAnnotation`，页面没有真正挂载标注组件。前后端不存在标注事件、权限校验或历史恢复逻辑，因此标注既无法正常使用，也无法同步给其他参与者。

视频和共享页面当前主要依赖初次加载的群成员数据以及零散的加入、离开事件更新 socket 信息，没有一份以服务端媒体房间为准的参与者快照。成员退出、断线、重连或重复收到事件时，页面无法保证参与状态准确。

## 目标

- 只有当前屏幕共享者可以绘制、撤销和清空标注。
- 当前共享中的其他参与者可以实时看到标注，但不能编辑。
- 后加入共享的成员可以恢复本次共享已有的全部标注。
- 标注与当前屏幕共享会话绑定，共享结束后立即清空。
- 视频页和共享页显示全部群成员，并分别标记其是否正在参与当前视频或共享。
- 加入、退出、断线和重连后，参与状态以服务端房间快照为准。
- 保持现有群组视频、屏幕共享和移动端布局不受破坏。

## 标注架构

### 服务端权威状态

标注历史由服务端管理，并保存在 Redis。每次屏幕共享会话使用 `groupId` 和现有 `startedAt` 组成独立的标注状态键，防止固定频道名称导致上一次共享的标注泄漏到下一次共享。

服务端只接受满足以下条件的写操作：

- 用户已经通过 Socket 鉴权。
- 用户已经加入对应群组。
- 当前屏幕共享会话仍然存在。
- 操作者是当前屏幕共享会话的 `ownerUserId`。

观看者发送的绘制、撤销或清空请求会被拒绝，不广播也不写入 Redis。

### 数据模型

每个已完成的标注动作具有稳定唯一的 `actionId`，并包含：

- 工具类型：画笔、箭头、矩形、文字或橡皮。
- 相对坐标点，取值范围为 `0` 到 `1`。
- 颜色和相对线宽。
- 可选文字内容。
- 创建者 `userId`。
- 创建时间。

使用相对坐标可以让不同分辨率和不同容器尺寸的客户端在同一屏幕位置绘制标注。

### 实时绘制流程

共享者开始绘制后，客户端发送带有 `actionId` 的临时绘制事件。移动过程中最多每 50 毫秒批量发送一次新增点，结束时发送完整动作。

服务端将临时事件广播给同一屏幕共享房间中的其他成员，但只在动作完成时将完整动作追加到 Redis。这样其他人可以看到实时轨迹，同时后加入者只恢复完整、稳定的历史，不会恢复半笔内容。

客户端按 `actionId` 合并临时更新，避免网络重发造成重复轨迹。动作完成后用服务端确认的完整动作替换临时轨迹。

### 撤销与清空

- 撤销只撤销共享者最近完成的一笔标注。
- 清空删除当前共享会话的全部标注。
- 两种操作都先经过服务端权限校验，再更新 Redis 并广播结果。
- 屏幕共享结束时，服务端删除本次会话的标注历史并广播清空事件。

### 后加入者恢复

成员加入屏幕共享媒体房间后，服务端随房间状态发送当前标注历史快照。客户端先按快照重建画布，再接收后续实时事件。

标注快照与屏幕共享房间绑定，不向只停留在群聊页面、尚未进入共享的成员发送。

## 前端组件职责

### `ScreenAnnotation`

组件继续负责画布绘制和工具交互，但本地历史改为由外部传入的权威动作列表驱动。

- 共享者模式：显示工具栏并允许产生绘制命令。
- 观看者模式：画布只读，不拦截页面操作。
- 组件向父页面发送绘制开始、增量、完成、撤销和清空意图。
- 组件接收动作快照和临时动作，并负责去重与重绘。
- 组件卸载时清理 resize、pointer 监听及未完成的临时状态。

保留现有注释、工具类型和视觉风格，只调整必要的数据流与事件接口。

### `groupScreen`

- 在共享视频上方真正挂载透明标注层。
- 只有 `isSharing` 为真时显示“开启标注”入口和工具栏。
- 观看者在有标注数据时自动渲染只读标注层。
- 绑定和解绑标注 Socket 事件。
- 进入共享后请求或接收历史快照。
- 离开页面或共享结束时清理本地标注状态。

## 实时参与状态

### 服务端房间快照

视频和共享媒体房间分别维护当前 socket 集合。服务端在以下场景重新生成并广播参与者快照：

- 成员加入媒体房间。
- 成员主动离开媒体房间。
- Socket 断开。
- 断线后重新鉴权并重新加入。
- 当前会话结束。

快照以唯一 `userId` 去重。若同一用户存在多个连接，只要仍有一个连接在房间中，该用户仍为“参与中”；最后一个连接离开后才变为“未参与”。

快照包含 `groupId`、会话类型、频道标识和当前参与用户 ID 列表。客户端拒绝不属于当前群组、当前会话类型或旧频道的快照。

### 客户端展示

群成员基础信息继续来自现有群组详情接口，参与状态来自媒体房间快照，两者不混为一份可变数据。

- 视频页使用视频房间的参与用户 ID 集合。
- 共享页使用共享房间的参与用户 ID 集合。
- 全部群成员均展示，并标记“参与中”或“未参与”。
- 列表使用 `member.id` 作为稳定 key。
- 会话结束或断线时清空对应参与集合，避免保留过期状态。

## Socket 事件

在现有 meeting Socket 中增加语义明确的事件：

- 标注临时更新。
- 标注动作完成。
- 标注历史快照。
- 标注撤销。
- 标注清空。
- 媒体房间参与者快照。
- 标注权限或会话失效错误。

所有监听器使用命名处理函数注册，并在页面销毁时用同一函数引用解除，避免重复监听。服务端不信任客户端传入的共享者身份，而是查询当前群组屏幕共享会话进行校验。

## 错误与边界处理

- Redis 写入失败时，不广播动作完成确认，并向共享者显示明确错误。
- 会话已经结束时，忽略迟到的绘制事件并通知客户端清空。
- 临时动作未正常结束时，不进入历史；客户端在断线或收到新快照时清除它。
- 重复的动作完成事件按 `actionId` 去重。
- 单次共享最多保存 1000 个完整动作，超限时拒绝新动作并提示共享者先清空。
- 单个动作最多包含 5000 个坐标点，文字最多 500 个字符；颜色只接受十六进制颜色，坐标与相对线宽必须处于约定范围，避免异常数据占用 Redis 或影响其他客户端。

## 测试设计

### 服务端测试

- 共享者完成标注后，同一共享房间的其他成员收到动作。
- 非共享者无法绘制、撤销或清空。
- 完成动作写入 Redis 状态，重复 `actionId` 不重复保存。
- 后加入者收到已有标注历史。
- 撤销删除最近一笔并广播。
- 清空删除全部历史并广播。
- 共享结束后删除标注状态，新共享不会恢复旧标注。
- 视频和共享参与者快照分别维护，互不污染。
- 同一用户多个 socket 只出现一个用户 ID。
- 最后一个 socket 离开后用户才变为未参与。
- 断线和重连后快照正确。

### 前端单元与组件测试

- 共享者可以绘制并发出规范化动作。
- 观看者只能渲染，不能产生写事件。
- 相同 `actionId` 的实时和完成事件只展示一笔。
- 标注快照可以恢复画布历史。
- 撤销、清空和会话结束正确更新画布。
- 组件卸载后不再响应 Socket 或窗口事件。
- 视频页和共享页将全部群成员与参与用户 ID 正确合并。
- 加入、退出、断线和重连快照更新参与标签。

### 页面与构建检查

- 两个浏览器上下文进入同一共享时，只有共享者能绘制，观看者实时可见。
- 观看者后加入时能看到此前标注。
- 共享结束并重新发起后不显示旧标注。
- 手机与桌面宽度下标注层和工具栏不横向溢出。
- 执行相关单元测试、组件测试、端到端测试、前后端类型检查和生产构建。

## 非目标

- 本次不引入独立白板、标注文件导出或跨共享会话永久保存。
- 本次不允许观看者参与绘制。
- 本次不替换现有 WebRTC 架构或引入 SFU。
- 本次不做与标注及参与状态无关的页面重构。
